Morgan Stanley - Summer Technology Analyst (June-August 2019)

Morgan Stanley generates hourly account position data was in a single line 1.5 GB sized file. This format of the file makes it rather time consuming to process, and it was imperative to reduce this time because often times, there would be swings in position values due the asset re-allocations or sometimes just system glitches. Because the file processing took so long, these swings could sometimes go unnoticed for several hours or days. Thus, during my three monnths, I developed OIRS: Optimized Information Retrieval System which processed these hourly files using Apache Hadoop and stored the position values in an easily query-able manner. Furthermore, I built a Position Swing Detector App that would allow users to easily identify significant differences in account position values by querying historical data stored. This system reduced the time by ~ 5 hours needed to identify abnormal swings in account position values.
